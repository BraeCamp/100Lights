# 100Lights Desktop v0.4.1

First desktop release since v0.3.0. (v0.4.0 was tagged before the final desktop
fixes landed and was never published — its draft can be deleted.)

## Desktop app

- **Computer Audio recording, no dialog** — true macOS system-audio loopback
  (macOS 13+). Recording computer audio no longer shows a screen-picker; macOS
  asks once for Screen & System Audio Recording permission.
- **Electron 34 → 38** — current runtime, security fixes, and the loopback
  capture above.
- **Resizable everywhere** — the launcher resizes from any corner and supports
  fullscreen (green traffic light works); all windows drag-resize from corners.
- **Fullscreen polish** — header padding for the traffic lights clears when
  macOS hides them in fullscreen.
- **Offline splash** — when the app can't reach the server it shows a retry
  screen and reconnects automatically instead of sitting blank.
- **Window model** — modules navigate inside the launcher window; projects and
  New Project open in their own window; "← Home" returns to the launcher.
- **Mic recording reliability** — permission diagnostics and error messages
  when input access fails.

## In the editor (ships via the web app, listed for context)

- MIDI overhaul: piano-roll editing, drum lanes, hardware MIDI input, 30s
  Capture buffer
- Spectral editor: edit clips as a time × frequency image (brush, wand,
  layers, de-noise)
- Help menu (H or ?) with searchable shortcuts and feature highlights
- Offline autosave and session restore
- Recording overhaul: mic-to-track, input meters, arm warnings
- 30+ DAW additions: varispeed, FM/wavetable synths, mixer EQ, WAV export,
  ripple editing, clip gain, transient split

## Smoke test after install

1. Launcher: drag any corner to resize; green traffic light enters fullscreen.
2. Open a project fullscreen — no dead padding under the notch/menu bar.
3. Audio editor → add track → input: Computer Audio → record — captures
   without a screen-picker dialog (one-time macOS permission prompt is OK).
4. Quit while offline → relaunch → offline splash appears, retries when back.
